# Отчет: FilmBot — Интелигентен асистент за филми с Weaviate Agents

## 1. Цел на проекта

Целта на проекта е разработването на интелигентно приложение **FilmBot**, което позволява на потребителите да задават въпроси на естествен език върху база данни от филми и жанрове, съхранени в **Weaviate Cloud**. Приложението използва **Query Agent** за семантично търсене и отговаряне на сложни заявки, и **Personalization Agent** за персонализирани препоръки базирани на потребителски профил.

---

## 2. Архитектура

```
Потребител
    │
    ▼
Streamlit UI (app.py)
    │
    ├──► Query Agent ──► Weaviate Cloud
    │         │              ├── Movies (50 записа)
    │         └──────────────└── Genres (12 записа)
    │
    └──► Personalization Agent ──► Movies (reference collection)
              │
              └── Persona + Interactions → Препоръки
```

### Компоненти

| Файл | Роля |
|------|------|
| `app.py` | Streamlit интерфейс — чат, sidebar, персонализация |
| `agent.py` | Query Agent и Personalization Agent логика |
| `data_loader.py` | Дефиниране на схемата и зареждане на данни |
| `config.py` | Константи, system prompt, demo заявки |
| `data/movies.json` | 50 филма с пълни метаданни |
| `data/genres.json` | 12 жанра с описания и характеристики |

---

## 3. Използвани агенти

### 3.1 Query Agent

Query Agent е pre-built услуга на Weaviate, която:
- Приема въпрос на естествен език
- Автоматично определя кои колекции да използва
- Генерира и изпълнява подходящи заявки (векторно търсене, BM25, филтри, агрегации)
- Поддържа контекст за follow-up въпроси
- Връща разбираем отговор на естествен език

Конфигуриран е с персонализиран system prompt, който го насочва да действа като "FilmBot" — ентусиазиран асистент за кино.

### 3.2 Personalization Agent

Персонализационният агент:
- Поддържа постоянни потребителски профили (персони)
- Записва взаимодействия (гледани/харесани филми) с тегло
- Извършва reranking на резултати спрямо потребителския профил
- Данните са персистентни между сесиите

## 4. Данни

### Movies — 50 филма
Обхваща периода 1954–2022, множество жанрове и националности. Всеки запис включва: заглавие, описание (векторизирано), година, IMDb рейтинг, режисьор, държава, език, жанр, продължителност.

### Genres — 12 жанра
Drama, Sci-Fi, Crime, Action, Thriller, Animation, Horror, Comedy, Romance, War, Western, Fantasy. Всеки жанр има: описание, типични теми, популярни десетилетия, среден рейтинг, настроение.

---

## 5. Примерни входове и изходи

### Заявка 1 — Обикновено семантично търсене
**Вход:** "Покажи ми sci-fi филми с рейтинг над 8.5"

**Изход:** FilmBot изброява Inception (8.8), Interstellar (8.7), The Matrix (8.7), Alien (8.5) с кратко описание на всеки.

---

### Заявка 2 — Multi-collection
**Вход:** "Кои жанрове имат най-висок среден рейтинг и какви филми от тях имаме?"

**Изход:** Агентът заявява едновременно Genres (за avg_rating) и Movies (за примерни заглавия), и синтезира отговор: Crime (8.5) → The Godfather, Pulp Fiction; Drama (8.3) → Shawshank, Fight Club; Western (8.4) → Django, Once Upon a Time...

---

### Заявка 3 — Follow-up
**Вход:** (след заявка 1) "А от тях кои са режисирани от Christopher Nolan?"

**Изход:** Агентът запазва контекст и отговаря само за Nolan: Inception (2010), Interstellar (2014).

---

### Заявка 4 — Агрегация
**Вход:** "Колко филма имаме с рейтинг над 8.0 и кои са от тях?"

**Изход:** Агентът изпълнява агрегираща заявка и изброява 30+ филма с рейтинг ≥ 8.0.

---

### Заявка 5 — Свободна формулировка
**Вход:** "Нещо напрегнато за гледане в петък вечер — нещо тъмно и интензивно"

**Изход:** Агентът извършва семантично търсене по настроение и препоръчва: The Dark Knight, Parasite, No Country for Old Men, Se7en (ако е в базата).

---

## 6. Алгоритми за обработка

- **Векторизация:** text2vec-openai (OpenAI Embeddings) — описанията на филмите и жанровете се превръщат в dense vectors
- **Семантично търсене:** nearText заявки за намиране на семантично сходни записи
- **BM25:** за точно ключово търсене (заглавия, режисьори)
- **Филтри:** where клаузи за числови и текстови полета (рейтинг, жанр, година)
- **Персонализация:** weighted interaction history → reranked results

---

## 7. Ограничения и идеи за бъдещо развитие

### Ограничения
- Базата данни е статична (50 филма) — не се актуализира автоматично
- Персонализацията работи само с Movies колекцията
- Няма поддръжка на изображения (постери)
- Query Agent изисква активна OpenAI API връзка

### Идеи за развитие
- Интеграция с TMDB API за автоматично актуализиране на данните
- Добавяне на TV Series колекция
- Transformation Agent за автоматично генериране на резюмета на български
- Гласов интерфейс (speech-to-text)
- Експорт на препоръки като плейлист/списък за гледане
- Оценяване на филми директно от интерфейса (thumbs up/down → interaction)
